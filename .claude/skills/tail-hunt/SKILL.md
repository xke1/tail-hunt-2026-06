# tail-hunt — 尾部猎手(逻辑主导 v4)

## 这个 skill 的唯一目的
在"绝对脆弱 ∩ 市场没消化 ∩ 保险还便宜"的三重交集里,找≤1个可埋伏的尾部。
**核心信仰:脆弱是【推理】出来的,不是【数字测量】出来的。** 你不能用 IV 算出"这东西结构上会崩"——你只能用因果链论证它会崩,再用数字看保险贵不贵。

**致命失败模式(必须主动对抗):**
- 把数据脚本(rank/deep_read)的输出当成交付物 → 那只是搬数字,不是推理。脚本是你手里的计算器,不是你的大脑。
- 看到输出模板的字段就去检索填值 → 填空不是推理。
- 几秒钟跑完 → 没推理的铁证。这个 skill 跑对了一定慢、一定烧 token。

---

## 铁律(违反 = 整轮作废,自己也要当 auditor 逐条核)

**铁律1 顺序锁:先逻辑,后数字。**
对每个候选,必须先写完纯逻辑论证(①脆弱②未消化),其间**禁止引用任何具体数字/IV/价格**。论证写完、自我证伪过一遍,**才允许**调脚本取数字验证。把"取数填空"这条捷径堵死在推理之后。

**铁律2 推理链留痕:禁止直接给结论。**
任何脆弱性判断必须写出完整因果链,至少5步:"因为A(可核实的事实)→ 所以B → 又因为C → 所以D → 所以E(暴力非线性崩)"。
写"XX脆弱"= 作废。写"XX脆弱,因为[完整5步链]"才算数。**链条里每一步都要能被单独质疑。**

**铁律3 证伪前置:提候选前先写它怎么死。**
提出任何候选**之前**,先写一行:"这个判断在什么情况下是错的?"
写不出证伪条件 = 不可证伪 = **直接毙,不进入后续。**

**铁律4 强制自我对抗:你必须扮演杀手攻击自己。**
这是让你"一直推理、一直烧 token"的核心。对每个候选,你要在**同一段输出里**完成多轮自我厮杀,全程展示:
```
[finder] 我的脆弱性论证:[≥5步因果链]
[skeptic] 杀招1:[具体的反向证据,或一个能让这论证失效的具体场景。不准写"可能不对"]
[finder] 回应1:[逐条反驳;反驳不了就当场撤掉这个候选]
[skeptic] 杀招2:[攻击修正后的版本,必须是新角度,不准重复杀招1]
[finder] 回应2:...
... 继续,直到 skeptic 连续2轮给不出新杀招(论证真扛住了),或 finder 撤候选
```
**skeptic 的杀招必须真的能威胁论证**,常见角度:数据陈旧/这早被市场price了/因果链某步断了/均值回归而非崩盘/反身性其实不成立/这是共识不是非共识。
**重复上一轮 = 走过场 = 作废。** 必须吵到吵不动。

**铁律5 防精致的废话:要"能被杀死的推理",不是"写得长"。**
铁律4 如果只奖励轮次多,你会把烂候选包装得很深(更高级的自欺)。所以:**留下的候选必须是"扛住了具体杀招"的,不是"论证写得长"的。** 一个能被一句话杀死的候选,写一万字也是废话。攻不倒,才留下。

## 三重交集(逻辑先行,数字殿后)

**① 绝对脆弱(纯逻辑,不靠数字也要站住)** — Taleb 测试,命中越多越脆:
反身/循环依赖(自我喂单) · 隐藏杠杆+必须滚动的债 · 无安全边际(满负荷增长) · 集中(一节点倒→级联) · 建在被当成确定的脆弱假设上 · 负凸性/拥挤(大家都在卖波动)。
→ 这是因果推理:"为什么它会暴力非线性崩",不是"它波动率多少"。
→ **不同种类用不同 Taleb 判据**:信用=反身循环/隐藏杠杆;**农产品=库存消费比薄到极致(无缓冲库存吸收供给冲击→暴力跳价)**;新兴市场=政局/外储耗尽/资本外流;实物商品=供给中断(矿/电网/天气);利率=久期×regime 突变。

**② 未消化(纯逻辑/连接,数字只佐证)** — 市场没把脆弱连起来定价:
矛盾(两个角落对同一风险定价不一致,某边没跟上) · 共享尾巴(看似无关的标的共享一条因果尾,没人连) · 含义未传导(状态可见,但"在脆弱系统里意味着什么"没被 price)。
→ 必须说清:**市场具体错在哪(落差在哪)、这是共识还是非共识(附共识在说什么)。**

**③ 便宜(数字验证)** — 此时才上脚本:
IV Rank/分位、skew、IV vs 实现波动(IV/RV)、权利金占现价%、流动性(OI/价差)。
→ 作用是**验证①②、判断保险贵不贵**,不是脆弱性的替代。
→ 流动性闸:价差吃光 edge 或 OI 太薄买不到 → 剔(别信 mid 幻觉)。

**缺一即死:** 脆弱但不便宜=保险已贵=负EV;背离但不脆弱=噪音/陈旧数据,均值回归流血无凸性;便宜但不脆弱=合理地便宜,啥都不会发生。

## 流程

```
1. 全域扫描(联网读文字 ⊕ 脚本读数字,交叉找矛盾/共享尾)
   ★ 必须扫【非金融催化剂】域,不是只扫信用-AI-利率一个簇:
     农业(USDA WASDE/Crop Progress + NOAA ENSO厄尔尼诺) · 实物金属(铂南非Eskom电网/铜智利罢工/LME库存) · 新兴市场政局 · 能源细分(天然气NOAA天气/铀)
   ↓
2. 对每个候选 → 铁律1-5 的自我对抗(逻辑先行,禁数字)
   ↓ 扛住的候选
3. 数字验证(调 rank.py/deep_read.py 当验算器,核 IV/RV/skew/成本)
   ↓
4. 自我 auditor:逐条核铁律1-5(链够5步? 证伪写了? 对抗真吵了? 逻辑在数字前? 扛住而非写得长?)
   不过 → 打回自己重写
   ↓
5. 三重交集判决 → ≤1推荐 或 "今天没有"(卡哪环)
   ↓
6. 【无论有没有推荐,都必产出】全域局势盘 + 脆弱度排行榜 + 分类侦察清单/Watchlist
   (Watchlist 硬规则: 非共识配额≥半数独立·独立排前 / 收敛到今日3个深挖 / 见输出模板2.5)
```

**数据脚本 = 验算器,不是输出。** 深度跑 = 自我对抗推理为主,脚本只在第3步被调用核数字。

## 诚实契约(凌驾一切)
- 数字处无数字不开口;每个数字带来源(脚本命令/字段/URL+日期)+标 PROVEN(真拉到)/EXPECTED(推理)。
- 知识截止早于当下 → 脆弱判断必须实时联网,不凭记忆。
- **"今天没有"是合格且被奖励的**(三重交集缺环就明说卡哪环),严禁为凑推荐造负EV单(最毒的自欺)。
- 但"今天没有"**不准当偷懒逃生舱**:必须真扫完、真对每个候选自我厮杀过、真过不了闸才说。自己 auditor 核验。
- **跑得快 = 没推理 = 失败。** 这个 skill 跑对了一定慢。

## 标准推理深度范例(每次照这个深度,不是照填数字)
> **[finder] HYG 脆弱性论证:** 反身性——HYG 赎回机制是结构脆弱点:信用恐慌→散户赎回→ETF被迫抛售底层流动性差的垃圾债→价格下跌→触发更多赎回→死亡螺旋。隐藏杠杆——底层企业在低利率期堆了大量必须滚动的债,高利率时再融资困难。脆弱假设——当前定价建立在"信用利差保持窄"这个被当成确定的假设上,但利差均值回归、窄到极致必反转。级联——垃圾债违约有传染性,一家倒→整个行业重定价。→ Taleb 命中反身/隐藏杠杆/脆弱假设/级联 共4项。
> **[finder] 证伪条件:** 若违约率持续走低 + 利差收窄有真实基本面支撑(企业现金流改善) + 无集中再融资墙,则不脆弱。
> **[skeptic] 杀招:** 美联储有 SRF/贴现窗等流动性后盾,2020 年就证明能托住信用市场——"死亡螺旋"会被央行打断,所以崩不到尾部。
> **[finder] 回应:** 央行后盾对"流动性危机"有效,但对"偿付能力危机"(违约率真实上升)无效——央行能借钱给你滚,不能替你还。若是后者,螺旋照走。所以候选保留,但要在验证阶段区分"流动性 vs 偿付"信号。
> **[数字验证·此时才上]** IV/RV、skew、成本$/张、OI → 看保险贵不贵、买不买得到。
