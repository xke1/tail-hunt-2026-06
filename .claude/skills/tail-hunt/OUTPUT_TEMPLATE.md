# tail-hunt 输出模板 v3(逻辑主体 · 局势盘强制 · 自我对抗留痕)

**核心:逻辑论证占主体版面,数字占验证版面。** 脆弱性/未消化是【因果推理】不是【测量】。
**无论有没有推荐,第1节(局势盘)和第2节(脆弱度排行榜)都必须产出**——这是你要的"看清全局",不是只给一个推荐就完事。

输出顺序:0运行头 → 1全域局势盘 → 2脆弱度排行榜 → 3候选深度对抗 → 4今日结论 → 5诚实节。

## 2. 脆弱度排行榜(强制·每次必产)
**全标的按"脆弱度"排序的概览**——不是埋伏推荐,是"哪些值得拿去做深度逻辑论证"的入口。
rank.py 出 Top-N,但每行要加一列【初判:为什么它可能脆弱】(一句话,纯逻辑直觉,数字佐证)。

```
#  标的   种类      现价    IV/RV   skew   离52w高%   初判:为什么可能脆弱(一句话)
1  TLT    利率尾    ...     1.41   ...    ...        长债,利率反转的非线性受害者
2  HYG    信用尾    ...     1.86   ...    ...        垃圾债赎回螺旋(见范例论证)
...
```
**排行榜按【种类】分组小结:** 债/信用簇整体什么状态、银行簇、防御股簇、商品簇...各自一句话。让 Ke 看清"今天脆弱集中在哪个种类"。

## 2.5 分类侦察清单 / Watchlist(强制·每次必产·"该往哪看"的情报图)
**不走三重交集闸(那是给开仓推荐的)。这节只负责"该往哪看",把全景拆成你能上 IBKR 动手的分类作业。"今天没有"也必须产出这节,让 Ke 不空手。**

**三条硬规则(防退化):**
- **① 非共识配额**:每类标【是否因果独立于今日头号共识簇】。**独立的排前面,且独立类别 ≥ 一半。** 防止清单偷偷变成"头号共识的 N 个马甲"(如 AI基建/半导体/加密本质都是"AI资本开支"那条尾的变体)。
- **② 轻重边界**:这里每条"为什么"可以是**一句话**(它是入口不是结论)。但一旦某标的进第3节深度对抗,**必须展开 ≥5步因果链 + 证伪条件 + 独立skeptic杀验**,不准用这里的一句话糊弄过深度对抗。
- **③ 强制收敛**:~6类 × 每类1-2个 = **8-10个"扫一眼"**;末尾**强制收敛到"今天最值得深挖的3个"**(按 最不共识+最便宜+催化剂最独立 挑)。铺开20个不如收敛3个。

```
类别 | 独立于共识?{独立✓/共识簇变体✗/半独立} | 盯(1-2个) | 为什么(一句话脆弱直觉) | 看哪个到期 | 看啥数据
农产品   | 独立✓   | WEAT/CORN | El Nino天气尾, 市场当非因素     | 9-12月(罩生长季)     | IV Rank/skew + USDA WASDE + ENSO状态
长债     | 独立✓   | TLT       | 鹰派Fed+通胀反弹, 长债极静→非线性 | ~90-180天(罩2-3次FOMC) | IV/RV + Fed点阵 + CPI/PCE
天然气   | 独立✓   | UNG       | AI电力边际燃料 + 取暖季           | 冬季Nov-Jan          | IV skew + EIA库存 + NOAA天气
工业金属 | 独立✓   | CPER/PPLT | 铜冶炼减产 / 铂南非电网          | ~90-180天            | LME库存 + Eskom拉闸 + WPIC季报
AI/集中度| 共识簇变体✗ | SMH       | Mag7占34%, 节点级联             | ~90-180天            | 哪个AI名IV rank还低(别碰已贵的)
加密     | 半独立  | MSTR/COIN | 杠杆; 已重挫                     | ~90天                | IV(大概率已高)+ skew
```
**收敛 · 今日最值得深挖的3个**:列3个 + 为什么是这3个(最不共识+最便宜+催化剂最独立)。

**统一数据规格(看每个标的都按这两层)**:
- **期权(IBKR肉眼)**:52W IV Rank/Perc · IV/RV · skew · 目标价外strike的 bid/ask·中点占现价%·OI·delta(~10-15delta=经典便宜尾)
- **催化剂(非市场数据)**:见上表各类别源
- **到期默认**:~90天主档 + ~150-180天跑道;**绝不 <30天**;对齐催化剂引爆日期(WASDE/FOMC/取暖季/财报)

## 3. 候选深度对抗(主体·逻辑占版面·每个候选一个 block)

> 铁律1 顺序锁:①②纯逻辑先写完(**禁碰数字**)→ ⑤才上数字验证。
> 只有过了第2节排行榜初筛、且 finder 认为值得深挖的候选,才进这一节做完整对抗。

```
### [候选] 标的/篮子 — 连接类型{矛盾 | 共享尾巴}

① 脆弱性论证(主栏·多段·纯因果链·≥5步·无数字也要站住)
   [finder] 因为A(可核实事实)→所以B→又因为C→所以D→所以E(暴力非线性崩)
   - 反身/循环依赖: <是什么,怎么自我强化>
   - 隐藏杠杆/必须滚动的债: <...>
   - 无安全边际/集中/级联: <哪个节点倒→怎么传染>
   - 脆弱假设: <建在什么"被当成确定"的假设上,为什么会反转>
   → Taleb 命中哪几项

② 未消化论证(主栏·多段·纯连接·讲清"市场错在哪")
   - 矛盾: 文字面说X危险 / 数字面定价成没事 / 落差具体在哪
   - 或 共享尾巴: 看似无关的{A,B,C}共享同一因果尾,因果链=<...>
   - 共识还是非共识?(附证据:共识在说啥) → edge=落差在哪

③ 证伪条件(铁律3·提候选必写)
   这个判断在什么情况下是错的? <写不出=这个候选直接毙>

④ 自我对抗记录(铁律4·必须真厮杀·这里是烧token的核心)
   [finder] 论证:[引①]
   [skeptic] 杀招1:[具体反证/证伪场景,禁泛泛质疑]
   [finder] 回应1:[逐条反驳 or 当场撤候选]
   [skeptic] 杀招2:[新角度,禁重复杀招1]
   [finder] 回应2:...
   ...(吵到skeptic连续2轮无新杀招=扛住,或finder撤=被杀)
   结论: 扛住(最后扛住了什么) / 撤掉(被哪招杀的)

⑤ 数字验证(铁律1·此时才上·验证①②+判便宜)
   现价 | RV%(分位) | ATM-IV | OTM-IV | skew | IV/RV(<1=便宜) | 成本$/张 | OI
   幻觉检查: IV随价外是否单调;脏价标⚠️剔
   → 便宜? (IV/RV相对排名 + skew + 流动性闸)

⑥ 三重交集判决: ①脆弱✓? ②未消化✓? ③便宜✓? → 过几环
```

## 4. 今日结论
```
≤1 推荐: 标的/结构(禁裸卖,只买或定义风险价差)/到期/strike/成本$/最大亏损(定死)/三栏理由(脆弱论证摘要/未消化摘要/便宜验证)/触发条件/生死线
  或
"今天没有": 卡在三重交集哪环(①还是②还是③)+ 为什么(诚实)
   —— 但第1节局势盘 + 第2节排行榜已经给了 Ke 全局,"今天没有"不等于"今天白跑"
→ 写入 output/tail-hunt_YYYY-MM-DD_HHMM.md + 生死线账本
```

## 5. 诚实节
```
- PROVEN(真拉到的数) vs EXPECTED(推理出的) 分别哪些
- 哪些候选被自我对抗杀掉了(及被哪招杀的) ← 这证明真的对抗过,不是偷懒
- 数据弱点(yfinance远月脏价? 免费数据无干净IV历史分位?)
- 残留共识污染(哪些"脆弱"其实已是公开house view)
- IBKR交叉核对状态(无订阅则注明仅OpenBB,待Ke肉眼复核)
```
